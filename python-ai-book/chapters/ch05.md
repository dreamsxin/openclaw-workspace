# 第5章 API 调用与本地推理

> 云端还是本地？本章帮你做出选择，并把两种路径都跑通。

## 5.1 两种路径对比

```
云端 API:   你的代码 → 互联网 → 大型GPU集群 → 结果
本地推理:   你的代码 → 本地进程 → 本地GPU/CPU → 结果
```

| 维度 | 云端 API | 本地推理 |
|------|----------|----------|
| **启动成本** | 零（注册即用） | 高（需要硬件） |
| **运行成本** | 按 Token 付费 | 电费 |
| **模型质量** | 最强（GPT-4, Claude） | 中等（7B-70B） |
| **速度** | 快（取决于网络） | 取决于硬件 |
| **隐私** | 数据离开本地 | 完全本地 |
| **离线** | ❌ | ✅ |
| **适合场景** | 生产环境、原型开发 | 隐私敏感、学习研究 |

## 5.2 云端 API 实战

### OpenAI API

```python
from openai import OpenAI

client = OpenAI()

# 基本调用
response = client.chat.completions.create(
    model="gpt-4",
    messages=[
        {"role": "system", "content": "你是一个Python专家"},
        {"role": "user", "content": "解释列表推导式"}
    ],
    temperature=0.7,     # 随机性 (0-1)
    max_tokens=500,      # 最大输出长度
    top_p=0.9,           # 核采样
    frequency_penalty=0, # 频率惩罚
    presence_penalty=0,  # 存在惩罚
)
```

### Claude API

```python
import anthropic

client = anthropic.Anthropic()

response = client.messages.create(
    model="claude-sonnet-4-20250514",
    max_tokens=1024,  # 必须指定
    system="你是一个Python专家",  # 系统提示词在这里
    messages=[
        {"role": "user", "content": "解释列表推导式"}
    ],
    temperature=0.7,
)
```

### 参数详解

```python
# temperature: 控制随机性
# 0.0 → 确定性输出（适合代码生成）
# 0.7 → 平衡（适合一般对话）
# 1.0 → 高创造性（适合创意写作）

# max_tokens: 控制输出长度
# 100 → 简短回答
# 1000 → 详细解释
# 4096 → 长文档

# top_p: 核采样（与 temperature 配合）
# 0.1 → 只考虑最可能的词
# 0.9 → 考虑更多可能性
```

```bash

curl -fsSL https://ollama.ai/install.sh | sh

ollama pull qwen2.5:7b       # 通用对话

```

```python
from openai import OpenAI

# Ollama 兼容 OpenAI 接口
client = OpenAI(
    base_url="http://localhost:11434/v1",
    api_key="ollama"
)

response = client.chat.completions.create(
    model="qwen2.5:7b",
    messages=[
        {"role": "user", "content": "解释装饰器"}

print(response.choices[0].message.content)
```

## 5.4 统一接口封装

写一次代码，云端和本地都能跑：

```python
# code/ch05/unified_client.py
from openai import OpenAI
from dataclasses import dataclass
from enum import Enum


class Backend(Enum):
    OPENAI = "openai"
    CLAUDE = "claude"
    OLLAMA = "ollama"
    VLLM = "vllm"


@dataclass
class AIConfig:
    backend: Backend
    model: str
    base_url: str | None = None
    api_key: str | None = None


class UnifiedAI:
    """统一的 AI 客户端，支持多种后端"""

    def __init__(self, config: AIConfig):
        self.config = config

        if config.backend == Backend.OPENAI:
            self.client = OpenAI()
        elif config.backend == Backend.OLLAMA:
            self.client = OpenAI(
                base_url="http://localhost:11434/v1",
                api_key="ollama"
            )
        elif config.backend == Backend.VLLM:
            self.client = OpenAI(
                base_url=config.base_url or "http://localhost:8000/v1",
                api_key="not-needed"
            )
        else:
            raise ValueError(f"Use anthropic SDK for Claude: {config.backend}")

    def chat(self, message: str, system: str = "", **kwargs) -> str:
        messages = []
        if system:
            messages.append({"role": "system", "content": system})
        messages.append({"role": "user", "content": message})

        response = self.client.chat.completions.create(
            model=self.config.model,
            messages=messages,
            **kwargs
        )
        return response.choices[0].message.content


# 使用示例
if __name__ == "__main__":
    # 切换后端只需要改这里
    config = AIConfig(
        backend=Backend.OLLAMA,  # 改成 OPENAI 就是云端
        model="qwen2.5:7b"
    )

    ai = UnifiedAI(config)
    print(ai.chat("用一句话介绍Python"))
```

## 5.5 流式输出对比

```python
# OpenAI 流式
stream = client.chat.completions.create(
    model="gpt-4",
    messages=[...],
    stream=True
)
for chunk in stream:
    print(chunk.choices[0].delta.content, end="")

# Claude 流式
with client.messages.stream(
    model="claude-sonnet-4-20250514",
    max_tokens=1024,
    messages=[...]
) as stream:
    for text in stream.text_stream:
        print(text, end="")
```

## 5.6 性能优化

### 批量请求

```python
# 并发调用多个请求
import asyncio
from openai import AsyncOpenAI

async def batch_call(prompts: list[str]):
    client = AsyncOpenAI()

    tasks = [
        client.chat.completions.create(
            model="gpt-4",
            messages=[{"role": "user", "content": p}]
        )
        for p in prompts
    ]

    responses = await asyncio.gather(*tasks)
    return [r.choices[0].message.content for r in responses]

# 运行
prompts = ["什么是变量？", "什么是函数？", "什么是类？"]
results = asyncio.run(batch_call(prompts))
```

### 缓存机制

```python
import hashlib
import json

_cache = {}

def cached_call(client, messages, model, **kwargs):
    """简单缓存：相同输入直接返回缓存结果"""
    key = hashlib.md5(
        json.dumps(messages, sort_keys=True).encode()
    ).hexdigest()

    if key in _cache:
        return _cache[key]

    response = client.chat.completions.create(
        model=model, messages=messages, **kwargs
    )
    result = response.choices[0].message.content
    _cache[key] = result
    return result
```

## 本章小结

- 云端 API：方便、质量高、按量付费
- 本地推理：隐私好、离线可用、需要硬件
- 用统一接口封装，一行代码切换后端
- 流式输出、并发调用、缓存是常见优化手段
