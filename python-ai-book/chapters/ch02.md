# 第2章 AI 绑定概念

> 在写代码之前，先搞清楚我们到底在做什么。

## 2.1 什么是「AI 绑定」？

简单说：**让 Python 程序能够调用 AI 模型的能力。**

更准确地说，AI 绑定是你的 Python 代码与 AI 模型之间的桥梁：

```
┌─────────────┐     HTTP/gRPC      ┌─────────────┐
│  Python 程序  │ ──────────────────→ │  AI 模型     │
│             │ ←────────────────── │  (本地/云端)  │
└─────────────┘     JSON 响应       └─────────────┘
```

这个「桥梁」包含三个核心要素：

### 1. 请求构造

把你的问题翻译成模型能理解的格式：

```python
request = {
    "model": "claude-sonnet-4-20250514",
    "messages": [
        {"role": "user", "content": "用Python写一个快速排序"}
    ]
}
```

### 2. 传输层

通过 HTTP 发送请求到模型服务：

```python
response = client.messages.create(**request)
```

### 3. 响应解析

从模型返回的结果中提取有用信息：

```python
answer = response.content[0].text
```

## 2.2 两种绑定模式

### 模式一：云端 API（主流）

```
你的代码 ──→ 云端服务器 ──→ 大型 GPU 集群 ──→ 返回结果
```

**代表服务：**
- Anthropic Claude API
- OpenAI GPT API
- Google Gemini API

**优点：**
- 无需本地 GPU
- 模型质量最高
- 开箱即用

**缺点：**
- 需要网络
- 按量付费
- 数据离开本地

### 模式二：本地推理

```
你的代码 ──→ 本地 Ollama/vLLM ──→ 本地 GPU/CPU ──→ 返回结果
```

**代表工具：**
- Ollama（最简单）
- vLLM（高性能）
- llama.cpp（极简）

**优点：**
- 完全离线
- 数据不出本机
- 无 API 费用

**缺点：**
- 需要 GPU（或耐心等待 CPU 推理）
- 模型规模受限于硬件
- 需要自己管理模型

## 2.3 统一接口：让两种模式可切换

本书的核心理念之一：**写一次代码，云端和本地都能跑。**

OpenAI 的 API 格式已成为事实标准。大多数工具都兼容它：

```python
# 云端
client = OpenAI(api_key="sk-xxx")
response = client.chat.completions.create(
    model="gpt-4",
    messages=[{"role": "user", "content": "你好"}]
)

# 本地 Ollama（同样的代码结构）
client = OpenAI(base_url="http://localhost:11434/v1", api_key="ollama")
response = client.chat.completions.create(
    model="qwen2.5:7b",
    messages=[{"role": "user", "content": "你好"}]
)
```

**关键区别只有两行：`base_url` 和 `model`。**

这就是"绑定"的本质 —— 用统一的 Python 接口，连接不同的 AI 后端。

## 2.4 请求-响应的数据流

一次完整的 AI 调用：

```python
# 1. 构造请求
messages = [
    {"role": "system", "content": "你是一个Python专家"},
    {"role": "user", "content": "解释装饰器"}
]

# 2. 发送请求
response = client.chat.completions.create(
    model="claude-sonnet-4-20250514",
    messages=messages,
    temperature=0.7,    # 控制随机性 (0-1)
    max_tokens=1000,    # 最大输出长度
)

# 3. 解析响应
answer = response.choices[0].message.content
tokens_used = response.usage.total_tokens

print(answer)
print(f"消耗 Token: {tokens_used}")
```

### 响应结构详解

```json
{
  "id": "chatcmpl-xxx",
  "object": "chat.completion",
  "model": "claude-sonnet-4-20250514",
  "choices": [
    {
      "index": 0,
      "message": {
        "role": "assistant",
        "content": "装饰器是Python的语法糖..."
      },
      "finish_reason": "stop"
    }
  ],
  "usage": {
    "prompt_tokens": 25,
    "completion_tokens": 150,
    "total_tokens": 175
  }
}
```

## 2.5 核心概念速查

| 概念 | 说明 |
|------|------|
| **Token** | 模型处理文本的最小单位，约 0.75 个英文单词或 1-2 个汉字 |
| **Temperature** | 控制输出随机性，0 = 确定性，1 = 创造性 |
| **Max Tokens** | 单次回复的最大 Token 数 |
| **System Prompt** | 系统提示词，定义模型的角色和行为 |
| **Context Window** | 模型单次能处理的最大 Token 总数（输入+输出） |
| **Streaming** | 流式输出，逐字返回而非等全部生成完 |

## 2.6 为什么选择 Python？

Python 是 AI 绑定的首选语言，原因很简单：

1. **生态最好** — 所有主流 AI SDK 都有 Python 版本
2. **语法简洁** — 几行代码就能完成一次 AI 调用
3. **数据科学生态** — pandas、numpy、matplotlib 等无缝配合
4. **社区活跃** — 遇到问题基本都能找到答案

## 本章小结

- AI 绑定 = Python 代码与 AI 模型之间的桥梁
- 两种模式：云端 API（方便）和本地推理（隐私）
- 用统一接口实现云端/本地可切换
- 核心数据流：构造请求 → 发送 → 解析响应
